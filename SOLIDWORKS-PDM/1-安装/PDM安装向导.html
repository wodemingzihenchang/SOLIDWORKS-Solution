Q&A

Webview2

需要SQL更高版本

要在 Microsoft® SQL Server 2019 或 Microsoft® SQL Server Express 2019 上创建或升级文件库数据库，SOLIDWORKS® PDM 软件需要“SQL Server 2019 Cumulative Update 4”或更高版本。更新 SQL Server 安装，请执行以下步骤：

1.安装 SQL Server 2019 或 SQL Server Express 2019 RTM（“SP0”）。

2.从 Microsoft.com下载 最新 SQL Server Cumulative Update。

3.在包含 SQL 2019 安装的同一计算机上运行更新程序文件。

4.继续创建 PDM 库的流程。

（要运行较旧的 SQL Cumulative Update（至少为“CU4”），您可以从 Microsoft 更新目录 (https://www.catalog.update.microsoft.com/Search.aspx?q=sql%20server%202019) 中选择其他内容）

ODBC问题

如果您安装或升级到 SOLIDWORKS® PDM 客户端版本 2020 或更高版本并且您的系统没有安装“Microsoft ODBC Driver 17 for SQL Server”，则您将无法访问文件库。您可能会遇到以下一个或多个问题：

• 无法访问数据库中的项目
• 登录提示无法显示，而是出现一个空的本地文件夹。
• 启动管理工具失败，并显示“由于未找到 msodbcsql17.dll，代码无法继续执行。重新安装程序可能会修复此问题”
• 创建新库失败并出现错误。
• 客户端日志可能会显示“未找到数据源名称，且未指定默认驱动程序”。
• 客户端日志可能会显示“[ODBC Driver Manager] Driver’s SQLAllocHandle on SQL_HANDLE_ENV failed”

方法：

要解决此安装问题，或修复“Microsoft ODBC Driver 17 for SQL Server”先决条件。您可以从安装文件夹安装该驱动程序：

或从 Microsoft 下载 ODBC Driver for SQL Server - ODBC Driver for SQL Server | Microsoft Learn
