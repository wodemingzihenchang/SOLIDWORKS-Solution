存档服务器找不到指定的ID文件

PDM 的后端有许多工具可用于对库中的错误进行故障诊断。如果遇到“存档服务器找不到具有指定 ID 的文件”或“访问存档服务器上的文件时出错”错误消息，这可能是由许多不同的来源引起的，因此此修复程序可能不适用于所有情况。

在对库进行更改期间发生错误时，可能会发生此特定情况。更改，例如修改安装、添加保管库、对服务器的更改或使用 SQL 数据库的新程序。

无论是什么原因导致了错误，当尝试检查库中的文件时，都会弹出以下错误消息：

  • “存档服务器找不到具有指定 ID 的文件”

或者，当尝试将文件从本地驱动器添加到保管库并收到此错误

  • “访问存档服务器上的文件时出错”

方法1检查存档位置

打开【菜单/程序-SOLIDWORKS PDM-存档服务器配置】工具。

导航到“存档”文件夹，然后找到导致错误的库。右键单击保管库，然后选择“属性”。在属性窗口中，有关于库存档位置路径的信息。

匹配存档工具中给出的路径，这应该可以修复错误。从 Windows 文件夹刷新PDM库，然后从“存档服务器”工具重新启动服务器。

测试更改：尝试将新文件上传到保管库并检入。检查存档服务器引用路径是否已恢复。

方法2检查日志

您可以从“SOLIDWORKS PDM 管理>日志文件“中访问有关这些错误的更多信息。可以收集有关例如服务器名称、数据库、错误模块等。

调查无法找到插件存档的原因：

• 确保路径有效，并且存档服务器服务可以访问该路径。如果存档位于网络共享上，您可能必须将存档服务器服务的登录帐户更改为具有存档文件夹完全访问权限的用户。
• 如果重新定位库存档至其他位置或驱动器后发生此错误，存档服务器服务可能在访问物理插件存档路径时遇到问题。
• 验证文件库存档中的“plugin”文件夹是否包含插件文件（每个插件都将具有包含两个名为“00000001.unk”和“index.xml”文件的子文件夹）。如果插件文件缺失，请通过管理工具将该插件从库中删除，然后再次添加。
• 如果库是复制的库，问题可能在于该插件无法复制，因此无法加载到客户端。请参阅知识库解决方案 S-046063。

如果在出于测试目的恢复文件库副本后遇到此错误：

• 如果您在该库中只恢复文件库数据库，则一切都可用，但存储在存档服务器上的物理存档文件除外。这包括任何已添加到库中的插件。如果原始插件存档缺失，请移除缺少的插件，方法是打开管理工具，登录库，展开插件节点并查看恢复的库中提供了哪些插件。右键单击并选择“移除”，以移除您没有的插件。
• 要恢复该库并保留原始插件，请从原始存档服务器的文件夹“…\0\Plugins\”下面复制插件存档。将它们复制到匹配的已恢复存档结构。
如果缺少的插件为“SWTaskAddIn”（即任务插件），并且想要在恢复的库副本中保留现有配置任务，则不要移除插件。而是在管理工具中打开“c:\program files<安装文件夹>\default data\convert_gb.cex”。将“swtaskaddin”拖放到已恢复的库的插件节点中。错误将显示未找到原始插件文件。关闭错误，任务插件将被更新为导入的副本。
• 如果缺少的插件是“Dispatch”，则移除“Dispatch”插件，然后从“c:\program files<安装文件夹>\default data\dispatch.caf”中再次添加。
