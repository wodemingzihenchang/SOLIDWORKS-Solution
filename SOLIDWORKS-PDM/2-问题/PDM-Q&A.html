BR10000410974检入失败

Check in of large drawing with many references can cause “an unspecified error occurred” with log showing a query resource error from SPU_UnlockFilesInBatch 2025 regression

Seen with SOLIDWORKS PDM 2025 SP4 and SP4, Windows 2022, SQL 2022 CU7

多选文件检入时，因查询错误而失败。客户端日志显示有三个错误依次出现：

--

Description: Exec SPU_UnlockFilesInBatch
Module: Explorer.EXE
Function: SpUnlock::mhUnlockBatch
Error code: E_UNEXPECTED
Error code general description: An unspecified error occurred.

--

Description: Caught error 1
Module: Explorer.EXE
Function: SpUnlock::mhUnlock
Error code: E_EDM_DATABASE_ACCESS
Error code general description: Could not access the item in the database.

--

Description:
Error running query in SpDbOdbc
State=37000
Msg=[Microsoft][ODBC Driver 17 for SQL Server][SQL Server]The query processor ran out of internal resources and could not produce a query plan. This is a rare event and only expected for extremely complex queries or queries that reference a very large number of tables or partitions. Please simplify the query. If you believe you have received this message in error, contact Customer Support Services for more information.

…

Begin Transaction
Exec SPU_UnlockFilesInBatch 1

Module: Explorer.EXE
Function: hGetErrCode
Error code: E_EDM_DATABASE_ACCESS

Workaround:
The following registry key avoids the check in error.

在检入失败的客户端：

关闭SOLIDWORKS，从通知托盘图标退出SOLIDWORKS PDM。添加注册表值：

密钥：ParallelCheckInUpload
类型：DWORD
值：0
